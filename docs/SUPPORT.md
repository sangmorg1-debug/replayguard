# Support process

Support requests should include ReplayGuard version, operating system, command or API route, sanitized error, and a minimal reproduction. Never include API keys, raw customer traces, secrets, or regulated data.

Priority targets:

- P0: confirmed security incident, cross-tenant exposure, or total production outage — immediate incident process.
- P1: major feature unavailable without workaround — acknowledge within one business day.
- P2: degraded behavior with workaround — acknowledge within two business days.
- P3: documentation, question, or feature request — triage during normal planning.

These are response targets until a staffed commercial support agreement defines enforceable service levels.
